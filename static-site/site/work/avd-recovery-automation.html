Azure Virtual Desktop recovery automation

I deployed and repaired Azure Virtual Desktop session hosts, then turned the repeated recovery steps into staged automation with discovery, deployment tracking, registration, and validation logs.

Context

AVD failures crossed Azure resources, virtual machines, domain join, host-pool registration, agents, audio redirection, profiles, and user-session behavior.

My role

I diagnosed the session-host problems, performed the repairs, and built PowerShell workflows to make the sequence more repeatable and observable.

Result

Session hosts could be deployed or recovered through explicit stages with resource discovery, status tracking, registration, and evidence for the final validation.

The problem

Why the work needed more than one tool

A session host can exist as a virtual machine and still fail as an AVD host. Domain identity, registration tokens, agents, host-pool state, profiles, audio settings, applications, and user-session behavior all have to align.

What could go wrong

  • Re-running a deployment against the wrong resource state
  • Treating VM power state as AVD health
  • Losing the registration or domain-join sequence
  • Changing profile or redirection settings without a validation record
  • Making scripts impossible to resume after a partial failure

What I did

The working sequence

  • Diagnosed domain-join failures, registration issues, agent health, audio redirection, and profile problems
  • Used Azure PowerShell and deployment scripts for repair and recovery
  • Added resource discovery rather than relying only on hard-coded assumptions
  • Separated deployment into staged execution modes
  • Tracked VM deployment and session-host registration state
  • Recorded validation output so a later run could identify what completed and what still failed

How I validated it

Evidence before completion

  • Confirmed Azure VM deployment state
  • Verified domain membership and required identity behavior
  • Checked host-pool registration and agent health
  • Tested audio redirection and profile behavior where they were part of the failure
  • Used logs to distinguish completed stages from work that needed to be resumed

What this demonstrates

The skill behind the tools

This project demonstrates how I turn one-off cloud troubleshooting into a safer operational workflow without hiding the real state behind a single script exit code.
