Experience

I learned by following problems across the stack.

I did not build my experience around one product or one layer. I kept following failures and migrations through identity, servers, storage, networking, cloud platforms, security controls, applications, and recovery.

The result is a broad systems view backed by hands-on work.

Infrastructure and modernization

I can make an inherited environment understandable

I’ve worked through aging servers, incomplete documentation, hidden dependencies, unsupported components, and changes that had to be sequenced around production risk.

  • Windows Server and Active Directory
  • DNS, DHCP, Group Policy, NTP, and certificates
  • File, print, application, and ERP dependencies
  • Current-state maps, runbooks, and staged modernization

Virtualization and storage

I treat the data path as part of the migration

I’ve moved and recovered workloads across VMware, Hyper-V, and Proxmox while accounting for SAN ownership, virtual disks, boot behavior, networking, and the source recovery path.

  • VMware ESXi and vCenter
  • Microsoft Hyper-V
  • Proxmox VE and Proxmox Backup Server
  • Dell PowerVault, SAS, iSCSI, LUNs, and multipath I/O
  • ReFS, NTFS, ZFS, VMFS, and virtual-disk conversion

Microsoft cloud and identity

I troubleshoot identity as an end-to-end system

I’ve followed authentication and application problems through Microsoft 365, Exchange Online, Entra ID, Azure, Azure Virtual Desktop, Windows hosts, profiles, tokens, and integrations.

  • Conditional Access and authentication
  • Mail flow, connectors, OAuth, and app registrations
  • Azure Virtual Desktop hosts, profiles, audio, and applications
  • OneDrive, SharePoint, Outlook, and Teams

Networking and security

I trace the complete path

I’ve designed and diagnosed segmented networks, remote access, published services, firewall policy, routing, NAT, DNS, TLS, monitoring, and the systems behind them.

  • Sophos firewalls

Backup and recovery

I want recovery evidence, not only successful jobs

I’ve worked with Veeam, Proxmox Backup Server, storage snapshots, restore validation, and recovery planning. I check whether the restored system can boot, authenticate, find its data, and perform its real function.

  • Backup architecture and troubleshooting
  • Restore tests and application validation
  • Source preservation during migrations
  • Rollback plans and continuity runbooks

Troubleshooting and automation

I use PowerShell, Linux shell tools, APIs, logs, registry data, packet and network evidence, and small automations to understand what a system is actually doing.

  • Failure-domain and root-cause analysis
  • PowerShell and shell-based diagnostics
  • API integration and operational scripting
  • Clear findings, runbooks, and handoff notes
