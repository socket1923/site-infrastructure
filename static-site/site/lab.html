Socket23 Lab

Build, break, measure, document

Socket23 is where I test infrastructure, security monitoring, local AI, and operational-agent ideas before turning the lessons into safer production work.

It is a controlled technical environment—not a substitute for production experience—and a place to study architecture, failure modes, recovery paths, integrations, and security boundaries directly.

Security Onion, Zeek, Elastic Stack, IDS and SIEM workflows, DShield honeypot data, enrichment, and detection validation.

Local AI

High-memory GPU systems, vLLM, llama.cpp, LM Studio, Ollama, Open WebUI, model APIs, local retrieval, and agent integrations.

Current research

A persistent, read-only Network Assistant

I am designing an assistant that helps explain infrastructure failures by correlating intended state, observed state, metrics, logs, runbooks, changes, and confirmed incident history.

The assistant can recommend actions and verify recovery through telemetry, but it does not hold production write credentials.

Security boundary

  • Read-only production access
  • Write access only to its own observation and knowledge stores
  • Human approval and execution for production changes
  • Independent alerts if the AI layer fails
  • Evidence and confidence attached to recommendations
